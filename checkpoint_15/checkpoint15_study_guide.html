AXIOS

Cliente HTTP basado en promesas para JavaScript

¿Qué es Axios?

Axios es una popular librería de JavaScript utilizada para realizar solicitudes HTTP desde el navegador o desde un entorno Node.js. Muchos proyectos en la web necesitan interactuar con una API REST en algún momento de su desarrollo, y axios nos brinda una interfaz sencilla y potente para interactuar con APIs y servicios web.

Características Principales

Cliente HTTP basado en promesas

Utiliza promesas de JavaScript para operaciones asincrónicas más limpias

Compatibilidad

Funciona tanto en navegador como en Node.js

Transformación automática

Convierte automáticamente datos de respuesta en formato JSON

Interceptores

Permite interceptar y modificar solicitudes y respuestas

Instalación y Uso Básico

1

Instalación:

npm install axios
2

Importación:

import axios from 'axios';

Ejemplo Práctico - Petición GET 

const [data, setData] = useState(null);

useEffect(() => {
  axios.get('https://api.example.com/data')
    .then(response => {
      setData(response.data);
    })
    .catch(error => {
      console.error('Error:', error);
    });
}, []);

Desglose detallado paso a paso:

1
Declaración del Estado
const [data, setData] = useState(null);

• data: Variable que almacenará la información recibida de la API
• setData: Función que permite actualizar el valor de data
• useState(null): Inicializa el estado con valor null (vacío)

2
Hook useEffect
useEffect(() => { ... }, [])

• Propósito: Ejecutar código cuando el componente se monta
• Array vacío []: Indica que solo se ejecute una vez
• Sin array: Se ejecutaría en cada render (evitar esto)

3
Petición GET con Axios
axios.get('https://api.example.com/data')

• GET: Método HTTP para obtener datos
• URL: Endpoint de la API que queremos consultar
• Retorna: Una promesa que se resolverá con la respuesta

4
Manejo de Respuesta Exitosa
.then(response => { setData(response.data); })

• then(): Se ejecuta cuando la petición es exitosa
• response: Objeto que contiene toda la respuesta del servidor
• response.data: Los datos reales que envió la API
• setData(): Actualiza el estado con los nuevos datos

5
Manejo de Errores
.catch(error => { console.error('Error:', error); })

• catch(): Se ejecuta cuando hay un error en la petición
• error: Objeto que contiene información del error
• console.error(): Registra el error en la consola del navegador
• Buena práctica: Siempre manejar errores en peticiones HTTP

6
Array de Dependencias
}, [])

• Array vacío []: El efecto solo se ejecuta una vez
• Con variables [count]: Se ejecuta cuando 'count' cambia
• Sin array: Se ejecuta en cada render (puede causar loops infinitos)
• Resultado: La petición HTTP se hace solo al montar el componente.
Este array se llama array de dependencias. Cuando está vacío ([]), le indica a React que el efecto (en este caso, la llamada axios.get(...)) solo debe ejecutarse una vez, justo después de que el componente se monte por primera vez en la interfaz.
Si el array no estuviera vacío...
- Si pusieras, por ejemplo, [userId], entonces el efecto se volvería a ejecutar cada vez que userId cambie.
- Si no pusieras ningún array, el efecto se ejecutaría en cada renderizado, lo cual normalmente no es deseado para llamadas HTTP, ya que estarías haciendo muchas peticiones innecesarias.

Otros Métodos HTTP con Axios

POST - Crear datos 
axios.post('/api/users', {
  name: 'Juan',
  email: 'juan@email.com'
})
.then(response => {
  console.log('Usuario creado:', response.data);
});
PUT - Actualizar datos 
axios.put('/api/users/1', {
  name: 'Juan Carlos',
  email: 'juan.carlos@email.com'
})
.then(response => {
  console.log('Usuario actualizado');
});
DELETE - Eliminar datos 
axios.delete('/api/users/1')
.then(response => {
  console.log('Usuario eliminado');
})
.catch(error => {
  console.error('Error al eliminar');
});
Con Headers de Autenticación 
axios.get('/api/protected', {
  headers: {
    'Authorization': 'Bearer tu-token-aqui',
    'Content-Type': 'application/json'
  }
});

React DevTools

Herramientas de depuración para aplicaciones React

¿Qué son las React DevTools?

Las React DevTools son una extensión de navegador que permite inspeccionar la jerarquía de componentes de React en las herramientas de desarrollador del navegador. Proporcionan widgets de inspección para ayudar en el desarrollo y depuración de aplicaciones React.

Dato importante: La depuración es una parte muy importante del desarrollo de software, permite detectar errores tempranamente y ser más eficiente.

Funcionalidades Principales

Inspección de Componentes

Ver estructura jerárquica, props y estado de componentes

Edición en Tiempo Real

Modificar props y estado directamente desde la herramienta

Análisis de Rendimiento

Profiler para registrar y analizar performance

Depuración Avanzada

Encontrar problemas y ver flujo de datos entre componentes

Event Listeners

Manejo de eventos en React para interfaces interactivas

¿Qué son los Event Listeners?

Un event listener es una función que se encarga de escuchar y responder a eventos específicos que ocurren en elementos del DOM o en objetos de JavaScript. En React, permiten que las aplicaciones respondan a las interacciones del usuario, haciendo que la interfaz sea dinámica e interactiva.

Finalizar toque

Ejemplos Prácticos

Ejemplo 1: Manejo de clics en botón

function Button() {
  const handleClick = () => {
    console.log('Botón clickeado');
  };

  return <button onClick={handleClick}>Haz clic</button>;
}

📝 Explicación: El event listener handleClick se ejecuta cuando se hace clic en el botón.

Ejemplo 2: Manejo de cambios en input

function InputField() {
  const [value, setValue] = useState('');

  const handleChange = (event) => {
    setValue(event.target.value);
  };

  return <input type="text" value={value} onChange={handleChange} />;
}

📝 Explicación: handleChange se activa cada vez que el usuario escribe, actualizando el estado.

Variables en SCSS

Potencia tu CSS con variables reutilizables y mantenibles

¿Qué son las Variables SCSS?

Las variables en SCSS (Sass) son una característica fundamental que permite almacenar valores reutilizables como colores, fuentes, tamaños y cualquier valor CSS. Se definen usando el símbolo $ y proporcionan una forma eficiente de mantener la consistencia y facilitar el mantenimiento del código CSS.

Ventaja clave: Cambia un valor en un lugar y se actualiza automáticamente en toda tu aplicación.

Sintaxis Básica 

// Definición de variables
$primary-color: #3498db;
$secondary-color: #e74c3c;
$font-size-base: 16px;
$font-family-main: 'Helvetica Neue', Arial, sans-serif;

// Uso de variables
.header {
  color: $primary-color;
  font-size: $font-size-base;
  font-family: $font-family-main;
}

.button {
  background-color: $secondary-color;
  padding: 10px 20px;
}

Tipos de Variables

Colores

$red: #dc3545;
$green: #28a745;

Tamaños

Breakpoints

$tablet: 1024px;

Organización de Variables 

// _variables.scss
// =====================
// COLORES
// =====================
$color-primary: #3498db;
$color-secondary: #2ecc71;
$color-accent: #e74c3c;

// =====================
// TIPOGRAFÍA
// =====================
$font-family-primary: 'Roboto', sans-serif;
$font-size-xs: 12px;
$font-size-sm: 14px;
$font-size-md: 16px;

// =====================
// ESPACIADO
// =====================
$spacing-xs: 4px;
$spacing-sm: 8px;
$spacing-md: 16px;
$spacing-lg: 24px;

Mejores Prácticas

1

Nombres Descriptivos

Usa nombres que describan el propósito, no la apariencia

// ❌ Mal
$red: #e74c3c;

// ✅ Bien
$color-danger: #e74c3c;
2

Variables por Defecto

Usa !default para variables que pueden ser sobrescritas

$primary-color: #3498db !default;
$base-font-size: 16px !default;
