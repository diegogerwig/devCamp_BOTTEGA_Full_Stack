¿Qué son las Variables?

Las variables en JavaScript son contenedores para almacenar valores de datos. Son fundamentales en cualquier lenguaje de programación, ya que permiten guardar y manipular información durante la ejecución de un programa.

Conceptos clave:

  • Declaración: Crear una variable con un nombre específico
  • Inicialización: Asignar un valor inicial a una variable
  • Reasignación: Cambiar el valor de una variable existente
  • Ámbito (Scope): Región del código donde una variable es accesible
  • Elevación (Hoisting): Comportamiento de las declaraciones en tiempo de ejecución

JavaScript ofrece tres formas principales de declarar variables: var, let y const. Cada una tiene comportamientos y casos de uso específicos.

Declaración con var

var es la forma tradicional de declarar variables en JavaScript, disponible desde las primeras versiones del lenguaje. Sin embargo, tiene algunas peculiaridades que pueden causar problemas en programas más complejos.

Características:

  • Ámbito de función o global, pero no de bloque
  • Se puede redeclarar y reasignar
  • Se eleva (hoisting) con valor inicial undefined

Ejemplos:

Ejemplo 1: Ámbito y redeclaración

// Declaración e inicialización
var nombre = "Ana";
console.log(nombre); // Ana

// Redeclaración (permitida con var)
var nombre = "Carlos";
console.log(nombre); // Carlos

// Ámbito de función
function ejemploVar() {
    var edad = 30;
    console.log(edad); // 30
}
ejemploVar();
// console.log(edad); // Error: edad is not defined

// Ámbito global vs bloque
var x = 10;
if (true) {
    var x = 20; // Misma variable que la x externa
    console.log(x); // 20
}
console.log(x); // 20 - ¡Ha cambiado el valor global!

Ejemplo 2: Hoisting

console.log(ciudad); // undefined (no error)
var ciudad = "Barcelona";
console.log(ciudad); // Barcelona

// El código anterior es interpretado como:
// var ciudad;
// console.log(ciudad); // undefined
// ciudad = "Barcelona";
// console.log(ciudad); // Barcelona

Nota importante:

Aunque var sigue siendo compatible, se recomienda usar let y const en el código moderno para evitar comportamientos inesperados y errores difíciles de detectar.

Declaración con let

let fue introducido en ES6 (ECMAScript 2015) para solucionar muchos de los problemas asociados con var. Proporciona un ámbito de bloque y un comportamiento más predecible.

Características:

  • Ámbito de bloque (dentro de llaves {})
  • No permite redeclaración en el mismo ámbito
  • Permite reasignación
  • Se eleva (hoisting) pero sin inicialización (zona muerta temporal)

Ejemplos:

Ejemplo 1: Ámbito de bloque

let x = 10;
if (true) {
    let x = 20; // Variable diferente a la x externa
    console.log(x); // 20
}
console.log(x); // 10 - No ha cambiado el valor global

// Otro ejemplo de ámbito
for (let i = 0; i < 3; i++) {
    console.log(i); // 0, 1, 2
}
// console.log(i); // Error: i is not defined

Ejemplo 2: Redeclaración y reasignación

let nombre = "Ana";
// let nombre = "Carlos"; // Error: no se puede redeclarar

// Pero sí podemos reasignar
nombre = "Carlos";
console.log(nombre); // Carlos

// En diferentes ámbitos sí es posible usar el mismo nombre
function ejemploLet() {
    let nombre = "Elena"; // Variable diferente
    console.log(nombre); // Elena
}
ejemploLet();
console.log(nombre); // Carlos

Ejemplo 3: Zona muerta temporal (TDZ)

// console.log(ciudad); // Error: Cannot access 'ciudad' before initialization
let ciudad = "Madrid";
console.log(ciudad); // Madrid

// A diferencia de var, las variables let se elevan pero no se inicializan
// y no se pueden usar antes de su declaración

Declaración con const

const también fue introducido en ES6 y está diseñado para declarar variables cuyo valor no cambiará durante la ejecución del programa (constantes).

Características:

  • Ámbito de bloque (igual que let)
  • No permite redeclaración ni reasignación
  • Debe inicializarse en la declaración
  • Se eleva (hoisting) sin inicialización (zona muerta temporal)
  • Para objetos y arrays, el contenido puede modificarse (es mutable)

Ejemplos:

Ejemplo 1: Declaración y uso básico

const PI = 3.14159;
console.log(PI); // 3.14159

// const TASA; // Error: Missing initializer in const declaration
// PI = 3.14; // Error: Assignment to constant variable

// Ámbito de bloque (igual que let)
if (true) {
    const PI = 3; // Diferente constante
    console.log(PI); // 3
}
console.log(PI); // 3.14159

Ejemplo 2: Objetos y arrays con const

// Objetos con const - La referencia es constante, pero el contenido es mutable
const persona = {
    nombre: "David",
    edad: 35
};

// No se puede reasignar el objeto
// persona = { nombre: "Laura", edad: 28 }; // Error

// Pero sí se pueden modificar sus propiedades
persona.nombre = "Laura";
persona.edad = 28;
console.log(persona); // { nombre: "Laura", edad: 28 }

// Arrays con const - Mismo comportamiento
const numeros = [1, 2, 3];
// numeros = [4, 5, 6]; // Error
numeros.push(4);
console.log(numeros); // [1, 2, 3, 4]

Ejemplo 3: Objetos inmutables con Object.freeze()

// Para hacer un objeto completamente inmutable, use Object.freeze()
const configuracion = Object.freeze({
    api: "https://api.ejemplo.com",
    timeout: 3000,
    debug: false
});

// En modo estricto, esto lanzaría un error
// En modo no estricto, simplemente no tiene efecto
configuracion.timeout = 5000; 
console.log(configuracion.timeout); // 3000 - No cambió

Comparación entre var, let y const

Característica var let const
Ámbito Función o Global Bloque Bloque
Redeclaración Sí No (en el mismo ámbito) No (en el mismo ámbito)
Reasignación Sí Sí No
Hoisting Sí, con valor undefined Sí, pero sin inicialización (TDZ) Sí, pero sin inicialización (TDZ)
Inicialización obligatoria No No Sí
Disponible desde Siempre ES6 (2015) ES6 (2015)

Buenas Prácticas

Preferir const por defecto

Utiliza const siempre que sea posible para variables que no necesiten ser reasignadas. Esto ayuda a prevenir errores y hace el código más predecible.

Usar let para variables que cambiarán

Cuando necesites reasignar valores, usa let en lugar de var. El ámbito de bloque de let hace el código más seguro y fácil de entender.

Evitar var en código moderno

En la mayoría de los casos, var puede ser reemplazado por let o const. Solo utiliza var si tienes una razón específica para hacerlo (como compatibilidad con navegadores muy antiguos).

Declarar variables al inicio de su ámbito

Declara todas las variables al principio del ámbito donde se utilizarán. Esto mejora la legibilidad y evita sorpresas con el hoisting.

Usar nombres descriptivos

Utiliza nombres que describan claramente lo que contiene la variable. Evita nombres de una sola letra excepto en casos muy específicos (como índices de bucles cortos).

// Incorrecto
const x = getUserData();

// Correcto
const userData = getUserData();
