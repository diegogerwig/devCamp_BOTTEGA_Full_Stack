Formularios en React

Aprende a manejar formularios de manera eficiente con componentes controlados y estado local

Select Dropdowns en React

Conceptos Clave:

  • Componente Controlado: El valor del select se maneja mediante el estado del componente
  • Prop value: Define qué opción está seleccionada actualmente
  • Evento onChange: Se ejecuta cada vez que el usuario cambia la selección
  • Sincronización: El estado y la UI siempre están sincronizados
  • Validación: Fácil implementación de reglas de validación
function SelectExample() {
  const [selected, setSelected] = useState('');
  
  return (
    <select value={selected} onChange={e => setSelected(e.target.value)}>
      <option value="">Selecciona una opción</option>
      <option value="react">React</option>
      <option value="vue">Vue</option>
    </select>
  );
}

Textarea en React

Diferencias con HTML tradicional:

  • Prop value: En React se usa value en lugar de contenido entre etiquetas
  • Consistencia: Mismo patrón que otros inputs (value + onChange)
  • Autosize: Fácil implementación de textarea que crece automáticamente
  • Validación en tiempo real: Contador de caracteres, límites, etc.
  • Formato automático: Aplicar transformaciones al texto
function TextareaExample() {
  const [text, setText] = useState('');
  
  return (
    <textarea
      value={text}
      onChange={e => setText(e.target.value)}
      placeholder="Escribe aquí..."
    />
  );
}

Comunicación Hijo → Padre

Patrón de Comunicación:

  • Callback Props: El padre pasa funciones como props al hijo
  • Lifting State Up: Elevar el estado al componente padre común
  • Event Bubbling: Los eventos se propagan hacia arriba
  • Data Flow: Flujo unidireccional de datos
  • Composición: Mejor que herencia para compartir lógica
// Padre
function Parent() {
  const [data, setData] = useState(null);
  
  return <Child onData={setData} />;
}

// Hijo
function Child({ onData }) {
  const sendData = () => onData('datos del hijo');
  
  return <button onClick={sendData}>Enviar</button>;
}

Manejo de Archivos con Dropzone

Implementa funcionalidad de arrastrar y soltar archivos de forma elegante y funcional

Instalación y Configuración

Características Principales:

  • Drag & Drop: Interfaz intuitiva para subir archivos

import { useDropzone } from 'react-dropzone';

function FileUploader() {
  const { getRootProps, getInputProps } = useDropzone({
    accept: { 'image/*': [] },

  });

    <div {...getRootProps()}>

    </div>
  );
}

Funcionalidades Avanzadas:

function AdvancedDropzone() {
  const [files, setFiles] = useState([]);
  
  const { getRootProps, getInputProps } = useDropzone({
    maxSize: 5242880, // 5MB
    onDrop: (accepted, rejected) => {
      setFiles(accepted);
      if (rejected.length) {
        alert('Algunos archivos fueron rechazados');
      }
    }
  });

  return (
    <div {...getRootProps()}>
      <input {...getInputProps()} />
      <p>Máximo 5MB por archivo</p>
    </div>
  );
}

Upload de Miniaturas

Optimización de Imágenes:

  • Canvas API: Redimensionar imágenes en el cliente
  • Compresión: Reducir tamaño de archivo manteniendo calidad
  • Formatos múltiples: Generar WebP, JPEG, PNG según necesidad
  • Lazy loading: Cargar miniaturas solo cuando son visibles
  • Caching: Almacenar miniaturas para evitar regeneración
function createThumbnail(file) {
  const canvas = document.createElement('canvas');
  const ctx = canvas.getContext('2d');
  const img = new Image();
  
  img.onload = () => {
    canvas.width = 150;
    canvas.height = 150;
    ctx.drawImage(img, 0, 0, 150, 150);
  };
  
  img.src = URL.createObjectURL(file);
}

Upload de Banner y Logo

Validaciones Específicas:

function validateImage(file, minWidth, minHeight) {
  const img = new Image();
  
  img.onload = () => {
    if (img.width < minWidth || img.height < minHeight) {
      alert(`Imagen muy pequeña. Mínimo: ${minWidth}x${minHeight}`);
    }
  };
  
  img.src = URL.createObjectURL(file);
}

Implementa operaciones completas de Crear, Leer, Actualizar y Eliminar datos

Manejador de Eliminación

Principios de Eliminación Segura:

  • Confirmación obligatoria: Siempre pedir confirmación antes de eliminar
  • Feedback visual: Mostrar estados de carga y resultado
  • Rollback capability: Posibilidad de deshacer eliminaciones
  • Error handling: Manejo robusto de errores de red/servidor
  • Optimistic updates: Actualizar UI inmediatamente
  • Batch operations: Eliminar múltiples elementos a la vez
function DeleteButton({ id, onDelete }) {
  const [isDeleting, setIsDeleting] = useState(false);
  
  const handleDelete = async () => {
    if (!confirm('¿Eliminar elemento?')) return;
    
    setIsDeleting(true);
    try {
      await onDelete(id);
    } finally {
      setIsDeleting(false);
    }
  };
  
  return (
    <button onClick={handleDelete} disabled={isDeleting}>
      {isDeleting ? 'Eliminando...' : 'Eliminar'}
    </button>
  );
}

Estrategias de Sincronización:

async function deleteItem(id) {
  const response = await fetch(`/api/items/${id}`, {
    method: 'DELETE'
  });
  
  if (!response.ok) {
    throw new Error('Error al eliminar');
  }
  
  return response.json();
}

function EditableItem({ item, onSave }) {
  const [isEditing, setIsEditing] = useState(false);
  const [value, setValue] = useState(item.name);
  
  const handleSave = () => {
    onSave({ ...item, name: value });
    setIsEditing(false);

Formulario de Edición

Características del Formulario:

  • Pre-población: Cargar datos existentes automáticamente
  • Validación client-side: Verificar datos antes de enviar
  • Dirty state tracking: Detectar cambios no guardados
  • Cancel functionality: Descartar cambios y volver al estado original
  • Loading states: Indicadores visuales durante el guardado
function EditForm({ item, onSave, onCancel }) {
  const [formData, setFormData] = useState(item);
  
  const handleSubmit = (e) => {
    e.preventDefault();
    onSave(formData);
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input 
        value={formData.name}
        onChange={e => setFormData({...formData, name: e.target.value})}
      />
      <button type="submit">Guardar</button>
      <button type="button" onClick={onCancel}>Cancelar</button>
    </form>
  );
}

Consultas Dinámicas a API

Técnicas de Optimización:

  • Debouncing: Retrasar consultas hasta que el usuario deje de escribir
  • Caching: Almacenar resultados para evitar consultas repetidas
  • Pagination: Cargar datos en páginas para mejorar performance
  • Infinite scroll: Carga progresiva de contenido
  • Query invalidation: Invalidar cache cuando los datos cambian
function useApiQuery(url, params) {
  const [data, setData] = useState([]);
  const [loading, setLoading] = useState(false);
  
  useEffect(() => {
    setLoading(true);
    fetch(`${url}?${new URLSearchParams(params)}`)
      .then(res => res.json())
      .then(setData)
      .finally(() => setLoading(false));
  }, [url, params]);
  
  return { data, loading };
}

Renderizado Condicional de Miniaturas

Estados de Imagen:

  • Loading state: Placeholder mientras carga la imagen
  • Error state: Imagen por defecto si falla la carga
  • Success state: Imagen cargada correctamente
  • Empty state: Cuando no hay imagen disponible
  • Progressive loading: Carga de baja a alta resolución
function SmartImage({ src, alt }) {
  const [status, setStatus] = useState('loading');
  
  return (
    <div>
      {status === 'loading' && <div>⏳ Cargando...</div>}
      {status === 'error' && <div>❌ Error al cargar</div>}
      <img 
        src={src} 
        alt={alt}
        onLoad={() => setStatus('loaded')}
        onError={() => setStatus('error')}
        style={{ display: status === 'loaded' ? 'block' : 'none' }}
      />
    </div>
  );
}

Corrección de Bugs en Edición

Problemas Comunes:

  • Memory leaks: No limpiar URLs de objeto creadas
  • State inconsistency: Desincronización entre cliente y servidor
  • Race conditions: Múltiples operaciones simultáneas
  • Error boundaries: Manejar errores sin romper la aplicación
  • Stale closures: Referencias obsoletas en callbacks
function ImageEditor() {
  const [images, setImages] = useState([]);
  
  // Limpiar URLs al desmontar
  useEffect(() => {
    return () => {
      images.forEach(img => {
        if (img.url.startsWith('blob:')) {
          URL.revokeObjectURL(img.url);
        }
      });
    };
  }, [images]);
  
  return <div>{/* UI */}</div>;
}

Eliminación Final de Imágenes

Implementación Robusta:

  • Optimistic updates: Actualizar UI inmediatamente
  • Rollback mechanism: Revertir cambios si falla la operación
  • Error recovery: Manejo inteligente de errores de red
  • Progress tracking: Mostrar progreso de operaciones largas
  • Batch processing: Procesar múltiples eliminaciones eficientemente
function useImageDeletion() {
  const [deleting, setDeleting] = useState(false);
  
  const deleteImage = async (id, optimisticUpdate) => {
    setDeleting(true);
    const rollback = optimisticUpdate(id);
    
    try {
      await fetch(`/api/images/${id}`, { method: 'DELETE' });
    } catch (error) {
      rollback();
      throw error;
    } finally {
      setDeleting(false);
    }
  };
  
  return { deleteImage, deleting };
}
