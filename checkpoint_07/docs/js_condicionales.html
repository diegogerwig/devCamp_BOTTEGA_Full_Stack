¿Qué es un Condicional?

Un condicional en programación es una estructura que permite ejecutar diferentes bloques de código dependiendo de si una condición se evalúa como verdadera o falsa. Los condicionales son fundamentales para la toma de decisiones en el código y controlan el flujo de ejecución de un programa.

Conceptos clave:

  • Condición: Una expresión que se evalúa como verdadera (true) o falsa (false)
  • Bloque de código: Conjunto de instrucciones que se ejecutarán si la condición se cumple
  • Alternativas: Diferentes rutas de ejecución dependiendo de diferentes condiciones
  • Operadores de comparación: Se utilizan para crear expresiones condicionales (==, ===, !=, !==, >, <, >=, <=)
  • Operadores lógicos: Combinan condiciones (&&, ||, !)

En JavaScript, hay varias formas de implementar condicionales, cada una con sus propios casos de uso y sintaxis.

Declaración if

La forma más básica de condicional en JavaScript es la declaración if. Permite ejecutar un bloque de código si una condición especificada es verdadera.

Sintaxis:

if (condición) {
  // código a ejecutar si la condición es verdadera
}

Ejemplos:

Ejemplo 1: Verificación de edad

const edad = 18;

if (edad >= 18) {
  console.log("Eres mayor de edad");
}
// Salida: "Eres mayor de edad"

Ejemplo 2: Verificación de usuario

const usuarioLogueado = true;

if (usuarioLogueado) {
  console.log("Bienvenido de nuevo");
}
// Salida: "Bienvenido de nuevo"

Condicionales en estructuras de datos

Filtrado de arrays

const numeros = [1, 5, 10, 15, 20, 25, 30];

// Filtrar números mayores que 15
const mayoresQue15 = numeros.filter(numero => {
  return numero > 15;  // condición para incluir en el resultado
});

console.log(mayoresQue15);  // [20, 25, 30]

// Filtrar usuarios activos
const usuarios = [
  { id: 1, nombre: "Ana", activo: true },
  { id: 2, nombre: "Carlos", activo: false },
  { id: 3, nombre: "Elena", activo: true },
  { id: 4, nombre: "David", activo: false }
];

const usuariosActivos = usuarios.filter(usuario => usuario.activo);
console.log(usuariosActivos);  // [{ id: 1, nombre: "Ana", activo: true }, { id: 3, nombre: "Elena", activo: true }]

Manejo de errores condicional

Try...catch...finally

Aunque no es un condicional tradicional, el bloque try...catch permite ejecutar código condicionalmente dependiendo de si ocurre un error.

function dividir(a, b) {
  try {
    // Se intenta ejecutar este código
    if (b === 0) {
      throw new Error("No se puede dividir por cero");
    }
    
    const resultado = a / b;
    return `Resultado: ${resultado}`;
  } catch (error) {
    // Se ejecuta solo si hay un error
    return `Error: ${error.message}`;
  } finally {
    // Se ejecuta siempre, haya error o no
    console.log("Operación finalizada");
  }
}

console.log(dividir(10, 2));   // "Resultado: 5"
console.log(dividir(10, 0));   // "Error: No se puede dividir por cero"

Declaración if...else

La declaración if...else permite ejecutar un bloque de código si la condición es verdadera y otro bloque diferente si la condición es falsa.

Sintaxis:

if (condición) {
  // código a ejecutar si la condición es verdadera
} else {
  // código a ejecutar si la condición es falsa
}

Ejemplos:

Ejemplo 1: Verificación de edad con alternativa

const edad = 16;

if (edad >= 18) {
  console.log("Eres mayor de edad");
} else {
  console.log("Eres menor de edad");
}
// Salida: "Eres menor de edad"

Ejemplo 2: Verificación de contraseña

const contraseña = "abc123";

if (contraseña.length >= 8) {
  console.log("Contraseña segura");
} else {
  console.log("Contraseña demasiado corta");
}
// Salida: "Contraseña demasiado corta"

Declaración if...else if...else

Esta estructura permite verificar múltiples condiciones en secuencia, ejecutando el bloque de código correspondiente a la primera condición que se evalúe como verdadera.

Sintaxis:

if (condición1) {
  // código a ejecutar si condición1 es verdadera
} else if (condición2) {
  // código a ejecutar si condición1 es falsa y condición2 es verdadera
} else if (condición3) {
  // código a ejecutar si condición1 y condición2 son falsas y condición3 es verdadera
} else {
  // código a ejecutar si todas las condiciones anteriores son falsas
}

Ejemplos:

Ejemplo 1: Calificaciones

const puntuacion = 85;
let calificacion;

if (puntuacion >= 90) {
  calificacion = "A";
} else if (puntuacion >= 80) {
  calificacion = "B";
} else if (puntuacion >= 70) {
  calificacion = "C";
} else if (puntuacion >= 60) {
  calificacion = "D";
} else {
  calificacion = "F";
}

console.log(`Calificación: ${calificacion}`);
// Salida: "Calificación: B"

Ejemplo 2: Estado del tiempo

const temperatura = 28;

if (temperatura < 0) {
  console.log("Temperatura bajo cero. ¡Abrígate mucho!");
} else if (temperatura < 10) {
  console.log("Hace frío. Necesitas un abrigo.");
} else if (temperatura < 20) {
  console.log("Temperatura fresca. Una chaqueta ligera es suficiente.");
} else if (temperatura < 30) {
  console.log("Temperatura agradable. Puedes salir en manga corta.");
} else {
  console.log("Hace calor. ¡No olvides hidratarte!");
}
// Salida: "Temperatura agradable. Puedes salir en manga corta."

Switch

La declaración switch evalúa una expresión y ejecuta el bloque de código correspondiente al valor que coincida. Es útil cuando tienes múltiples condiciones basadas en el mismo valor.

Sintaxis:

switch (expresión) {
  case valor1:
    // código a ejecutar si expresión === valor1
    break;
  case valor2:
    // código a ejecutar si expresión === valor2
    break;
  case valor3:
    // código a ejecutar si expresión === valor3
    break;
  default:
    // código a ejecutar si ninguno de los valores coincide
}

Nota importante:

La sentencia break es crucial en cada caso para evitar la "caída" (fallthrough) a los siguientes casos. Sin el break, el código seguirá ejecutándose en los casos siguientes independientemente de si coinciden o no.

Ejemplos:

Ejemplo 1: Días de la semana

const diaDeLaSemana = 3;
let nombreDia;

switch (diaDeLaSemana) {
  case 1:
    nombreDia = "Lunes";
    break;
  case 2:
    nombreDia = "Martes";
    break;
  case 3:
    nombreDia = "Miércoles";
    break;
  case 4:
    nombreDia = "Jueves";
    break;
  case 5:
    nombreDia = "Viernes";
    break;
  case 6:
    nombreDia = "Sábado";
    break;
  case 7:
    nombreDia = "Domingo";
    break;
  default:
    nombreDia = "Día inválido";
}

console.log(`Hoy es ${nombreDia}`);
// Salida: "Hoy es Miércoles"

Ejemplo 2: Tipo de fruta (con "fallthrough" intencional)

const fruta = "Manzana";
let tipo;

switch (fruta) {
  case "Manzana":
  case "Pera":
  case "Durazno":
    tipo = "Fruta de pepita";
    break;
  case "Naranja":
  case "Limón":
  case "Mandarina":
    tipo = "Cítrico";
    break;
  case "Plátano":
  case "Piña":
    tipo = "Fruta tropical";
    break;
  default:
    tipo = "Otro tipo de fruta";
}

console.log(`${fruta} es una ${tipo}`);
// Salida: "Manzana es una Fruta de pepita"

Operadores de Comparación y Lógicos

Los condicionales utilizan expresiones que se evalúan como verdaderas o falsas. Estas expresiones se construyen usando operadores de comparación y lógicos.

Operadores de Comparación

Operador Descripción
== Igual (valor)
=== Estrictamente igual (valor y tipo)
!= No igual (valor)
!== Estrictamente no igual (valor y tipo)
> Mayor que
< Menor que
>= Mayor o igual que
<= Menor o igual que

Operadores Lógicos

Operador Descripción
&& Y lógico (ambas condiciones deben ser verdaderas)
|| O lógico (al menos una condición debe ser verdadera)
! NO lógico (invierte el valor de la condición)

Ejemplos:

// Comparaciones
console.log(5 == "5");    // true (igualdad de valor)
console.log(5 === "5");   // false (diferentes tipos)
console.log(10 > 5);      // true
console.log(10 < 20);     // true
console.log(5 >= 5);      // true
console.log(7 <= 5);      // false

// Operadores lógicos
const edad = 25;
const tieneLicencia = true;

// AND lógico
if (edad >= 18 && tieneLicencia) {
  console.log("Puede conducir");  // Se ejecuta
}

// OR lógico
const esEstudiante = false;
const esMayorDe65 = false;

if (esEstudiante || esMayorDe65) {
  console.log("Descuento aplicable");  // No se ejecuta
}

// NOT lógico
const estaBloqueado = false;

if (!estaBloqueado) {
  console.log("Acceso permitido");  // Se ejecuta
}

Buenas Prácticas

Utilizar === en lugar de ==

El operador de igualdad estricta (===) compara tanto el valor como el tipo, evitando conversiones de tipos inesperadas.

// ❌ Puede llevar a comportamientos inesperados
if (valor == 0) { ... }

// ✅ Más seguro y predecible
if (valor === 0) { ... }

Usar llaves incluso para bloques de una sola línea

Por claridad y para evitar errores, es recomendable usar llaves incluso cuando el bloque condicional contiene una sola línea.

// ❌ Propenso a errores si se añaden más líneas después
if (condicion) console.log("Una línea");

// ✅ Más claro y menos propenso a errores
if (condicion) {
  console.log("Una línea");
}

Evitar anidaciones excesivas

Demasiados niveles de anidación pueden hacer que el código sea difícil de leer y mantener. Considera refactorizar usando funciones, operadores lógicos o retornos tempranos.

// ❌ Difícil de leer y mantener
function verificarAcceso(usuario) {
  if (usuario) {
    if (usuario.rol) {
      if (usuario.rol === 'admin') {
        return true;
      } else {
        return false;
      }
    } else {
      return false;
    }
  } else {
    return false;
  }
}

// ✅ Más legible con retornos tempranos
function verificarAcceso(usuario) {
  if (!usuario) return false;
  if (!usuario.rol) return false;
  return usuario.rol === 'admin';
}

Considerar el operador ternario para casos simples

Para asignaciones condicionales simples, el operador ternario puede hacer que el código sea más conciso.

// if/else tradicional
let mensaje;
if (edad >= 18) {
  mensaje = "Adulto";
} else {
  mensaje = "Menor";
}

// ✅ Más conciso con operador ternario
const mensaje = edad >= 18 ? "Adulto" : "Menor";

Casos de Uso Prácticos

Validación de formularios

function validarFormulario(usuario, contraseña) {
  // Verificar campos vacíos
  if (!usuario || !contraseña) {
    return "Todos los campos son obligatorios";
  }
  
  // Verificar longitud de contraseña
  if (contraseña.length < 8) {
    return "La contraseña debe tener al menos 8 caracteres";
  }
  
  // Verificar formato de usuario (solo letras y números)
  if (!/^[a-zA-Z0-9]+$/.test(usuario)) {
    return "El usuario solo puede contener letras y números";
  }
  
  // Todo correcto
  return "Formulario válido";
}

console.log(validarFormulario("", "abc123"));        // "Todos los campos son obligatorios"
console.log(validarFormulario("usuario", "123"));    // "La contraseña debe tener al menos 8 caracteres"
console.log(validarFormulario("usuario@", "password123")); // "El usuario solo puede contener letras y números"
console.log(validarFormulario("usuario", "password123"));  // "Formulario válido"

Control de acceso

function verificarPermisos(usuario) {
  // Si no hay usuario, no hay acceso
  if (!usuario) {
    return "Acceso denegado: Usuario no encontrado";
  }
  
  // Verificar si el usuario está activo
  if (!usuario.activo) {
    return "Acceso denegado: Usuario inactivo";
  }
  
  // Verificar el rol del usuario
  switch (usuario.rol) {
    case "admin":
      return "Acceso concedido: Acceso total";
    case "editor":
      return "Acceso concedido: Permiso de edición";
    case "usuario":
      return "Acceso concedido: Permiso de lectura";
    default:
      return "Acceso denegado: Rol desconocido";
  }
}

// Ejemplos
console.log(verificarPermisos(null));  // "Acceso denegado: Usuario no encontrado"
console.log(verificarPermisos({ activo: false, rol: "usuario" }));  // "Acceso denegado: Usuario inactivo"
console.log(verificarPermisos({ activo: true, rol: "admin" }));  // "Acceso concedido: Acceso total"
console.log(verificarPermisos({ activo: true, rol: "invitado" }));  // "Acceso denegado: Rol desconocido"

// Ejemplo con usuario activo y rol "editor"
const usuario = { activo: true, rol: "editor" };
console.log(verificarPermisos(usuario));  // "Acceso concedido: Permiso de edición"
