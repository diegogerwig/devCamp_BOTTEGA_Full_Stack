¿Qué es el Operador Ternario?

El operador ternario (también conocido como operador condicional) es el único operador en JavaScript que toma tres operandos. Es una forma abreviada de escribir una declaración condicional if-else simple en una sola línea.

Sintaxis:

condición ? expresión_si_verdadero : expresión_si_falso
  • condición: Expresión que se evalúa como verdadera o falsa
  • expresión_si_verdadero: Valor que se devuelve si la condición es verdadera
  • expresión_si_falso: Valor que se devuelve si la condición es falsa

El operador ternario se llama "ternario" porque tiene exactamente tres operandos, a diferencia de los operadores unarios (con un solo operando) o binarios (con dos operandos).

Ejemplos Básicos

Ejemplo 1: Verificar si es mayor de edad

// Usando if-else
const edad = 20;
let mensaje;

if (edad >= 18) {
  mensaje = "Eres mayor de edad";
} else {
  mensaje = "Eres menor de edad";
}

console.log(mensaje);  // "Eres mayor de edad"

// Usando operador ternario
const edadPersona = 16;
const mensajeTernario = edadPersona >= 18 ? "Eres mayor de edad" : "Eres menor de edad";

console.log(mensajeTernario);  // "Eres menor de edad"

Ejemplo 2: Determinar paridad de un número

const numero = 7;
const paridad = numero % 2 === 0 ? "par" : "impar";

console.log(`El número ${numero} es ${paridad}`);  // "El número 7 es impar"

Ejemplo 3: Seleccionar un mensaje de bienvenida

const horaDelDia = 14;  // 2 PM
const saludo = horaDelDia < 12 ? "Buenos días" : horaDelDia < 18 ? "Buenas tardes" : "Buenas noches";

console.log(saludo);  // "Buenas tardes"

Comparación con if-else

Utilizando if-else

function verificarAcceso(usuario) {
  let mensaje;
  
  if (usuario.activo) {
    if (usuario.rol === 'admin') {
      mensaje = 'Acceso total';
    } else {
      mensaje = 'Acceso limitado';
    }
  } else {
    mensaje = 'Acceso denegado';
  }
  
  return mensaje;
}

Utilizando operador ternario

function verificarAcceso(usuario) {
  return usuario.activo
    ? usuario.rol === 'admin'
      ? 'Acceso total'
      : 'Acceso limitado'
    : 'Acceso denegado';
}

// O en una sola línea (menos legible):
function verificarAcceso(usuario) {
  return usuario.activo ? (usuario.rol === 'admin' ? 'Acceso total' : 'Acceso limitado') : 'Acceso denegado';
}

Ventajas y Desventajas

Ventajas del Operador Ternario

  • Más conciso, menos verboso
  • Permite asignar valores en una sola línea
  • Puede usarse en expresiones
  • Útil para renderizado condicional en frameworks

Desventajas del Operador Ternario

  • Puede ser difícil de leer si se anida demasiado
  • No es adecuado para lógica compleja
  • Puede dificultar la depuración
  • Menos explícito que if-else en algunos casos

Casos de Uso Comunes

1. Asignación condicional de variables

// Asignar un valor predeterminado si una variable es nula o indefinida
const nombre = nombreUsuario ? nombreUsuario : 'Invitado';

// Versión más corta usando el operador de coalescencia nula (ES2020)
const nombre = nombreUsuario ?? 'Invitado';

2. Renderizado condicional en React

// Mostrar un componente diferente basado en si el usuario está autenticado
function App() {
  const [isLoggedIn, setIsLoggedIn] = useState(false);
  
  return (
    

      {isLoggedIn ?  : }
    

  );
}

3. Agregar clases CSS condicionalmente

// Agregar una clase si un elemento está activo
const buttonClass = isActive ? 'btn btn-active' : 'btn';

// Ejemplo con múltiples condiciones
const buttonClass = `btn ${isActive ? 'btn-active' : ''} ${isDisabled ? 'btn-disabled' : ''}`.trim();

4. En funciones de mapeo de arrays

// Transformar un array basado en condiciones
const números = [1, 2, 3, 4, 5];
const parOImpar = números.map(num => num % 2 === 0 ? 'par' : 'impar');

console.log(parOImpar);  // ["impar", "par", "impar", "par", "impar"]

Operador Ternario Anidado

Es posible anidar operadores ternarios para manejar múltiples condiciones, pero debemos tener cuidado con la legibilidad.

Ejemplo: Categorizar calificaciones

const puntuación = 85;

// Operador ternario anidado
const calificación = 
  puntuación >= 90 ? 'A' :
  puntuación >= 80 ? 'B' :
  puntuación >= 70 ? 'C' :
  puntuación >= 60 ? 'D' : 'F';

console.log(`Calificación: ${calificación}`);  // "Calificación: B"

⚠️ Advertencia: Cuidado con la legibilidad

Aunque el operador ternario anidado es poderoso, puede hacer que el código sea difícil de leer y mantener. Si tienes más de dos o tres niveles de anidación, considera usar una declaración if-else más tradicional o un objeto de mapeo.

// Alternativa usando un objeto de mapeo
function obtenerCalificación(puntuación) {
  const rangos = {
    90: 'A',
    80: 'B',
    70: 'C',
    60: 'D',
    0: 'F'
  };
  
  // Encontrar el primer umbral que la puntuación supera
  const umbral = Object.keys(rangos)
    .map(Number)
    .sort((a, b) => b - a)
    .find(umbral => puntuación >= umbral);
  
  return rangos[umbral];
}

Buenas Prácticas

Mantener la simplicidad

Usa el operador ternario para casos sencillos donde mejore la legibilidad. Para lógica compleja, prefiere if-else.

// ✅ Bueno: Simple y claro
const mensaje = edad >= 18 ? "Adulto" : "Menor";

// ❌ Malo: Demasiado complejo
const mensaje = edad >= 18 ? (genero === 'M' ? (casado ? "Hombre casado" : "Hombre soltero") : (casado ? "Mujer casada" : "Mujer soltera")) : "Menor de edad";

Usar paréntesis para claridad

Cuando anides operadores ternarios, usa paréntesis para hacer explícito el orden de evaluación.

// ❌ Difícil de leer
const resultado = a > b ? c > d ? e : f : g > h ? i : j;

// ✅ Más fácil de entender
const resultado = a > b 
  ? (c > d ? e : f) 
  : (g > h ? i : j);

Formatea adecuadamente

Usa saltos de línea y sangrías para mejorar la legibilidad, especialmente en operadores ternarios anidados.

// Una línea para expresiones muy simples
const esAdulto = edad >= 18 ? true : false;

// Múltiples líneas para expresiones más complejas
const mensaje = esFinDeSemana
  ? temperatura > 25
    ? "Vamos a la playa"
    : "Quedémonos en casa"
  : "Hay que trabajar";

Evitar efectos secundarios

El operador ternario está diseñado para devolver valores, no para ejecutar código con efectos secundarios.

// ❌ Mal uso con efectos secundarios
isLoggedIn ? updateUserStatus() : showLoginForm();

// ✅ Mejor con if-else
if (isLoggedIn) {
  updateUserStatus();
} else {
  showLoginForm();
}

Otras Alternativas al Operador Ternario

Operador de cortocircuito (&&)

Para ejecutar código solo si una condición es verdadera (sin parte "else").

// Usando if
if (condición) {
  ejecutarCódigo();
}

// Usando operador &&
condición && ejecutarCódigo();

Operador de coalescencia nula (??)

Introducido en ES2020, devuelve el operando derecho cuando el izquierdo es null o undefined.

// Usando operador ternario
const nombre = usuario.nombre !== null && usuario.nombre !== undefined ? usuario.nombre : 'Invitado';

// Usando operador de coalescencia nula
const nombre = usuario.nombre ?? 'Invitado';

Objetos de mapeo

Útil cuando tienes múltiples condiciones con valores diferentes.

// Usando if-else o ternarios anidados
let mensaje;
if (status === 'success') {
  mensaje = 'Operación exitosa';
} else if (status === 'warning') {
  mensaje = 'Advertencia: Continúe con precaución';
} else if (status === 'error') {
  mensaje = 'Error: Inténtelo de nuevo';
} else {
  mensaje = 'Estado desconocido';
}

// Usando un objeto de mapeo
const mensajes = {
  success: 'Operación exitosa',
  warning: 'Advertencia: Continúe con precaución',
  error: 'Error: Inténtelo de nuevo'
};

const mensaje = mensajes[status] || 'Estado desconocido';

Ejercicios Prácticos

Ejercicio 1: Convertir if-else a operador ternario

// Convierte este código a operador ternario
function validarPassword(password) {
  if (password.length >= 8) {
    return "Password válido";
  } else {
    return "Password demasiado corto";
  }
}

Solución:

function validarPassword(password) {
  return password.length >= 8 ? "Password válido" : "Password demasiado corto";
}

Ejercicio 2: Crear un semáforo

Escribe una función que tome un color ("rojo", "amarillo" o "verde") y devuelva el mensaje correspondiente.

function mensajeSemaforo(color) {
  // Usar operador ternario aquí
}

Solución:

function mensajeSemaforo(color) {
  return color === "rojo" ? "Detente" :
         color === "amarillo" ? "Precaución" :
         color === "verde" ? "Avanza" :
         "Color no válido";
}
